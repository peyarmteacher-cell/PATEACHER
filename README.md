# 🇹🇭 ระบบครู วPA (สพฐ.) - PHP & MySQL Database Service Web App

ยินดีต้อนรับสู่โปรเจกต์ระบบสารสนเทศประเมินตนเองและจัดทำข้อตกลง วPA ของข้าราชการครู สพฐ. ระบบนี้ได้รับการออกแบบมาอย่างสวยงาม ลื่นไหลด้วยเทคโนโลยี React + TypeScript ในส่วนของหน้ากากผู้ใช้ (Frontend) และใช้ตัวเชื่อมต่อข้อมูลประสิทธิภาพสูงด้วยภาษา PHP (PDO) ร่วมกับฐานข้อมูล MySQL (phpMyAdmin) ฝั่งเซิร์ฟเวอร์โรงเรียนของคุณครู

---

## 📌 สรุปหลักการทำงานของระบบ (สถาปัตยกรรม React + PHP)

เพื่อตอบคำถามเกี่ยวกับเรื่องโครงสร้างเว็บที่สร้างขึ้นมา:

1. **ฝั่งผู้ใช้งาน (Frontend):** 
   * พัฒนาด้วยเทคโนโลยี **TypeScript / React** เพื่อความลื่นไหลระดับสูง (Single Page Application)
   * **ไม่ต้องเป็นกังวล:** เวลาใช้งานจริงบนเซิร์ฟเวอร์โรงเรียน คุณครูไม่ได้รันไฟล์ TypeScript ดิบครับ! เมื่อเราทำการสั่งรันคำสั่งคอมไพล์ (`npm run build`) ตัว Vite จะทำการแปลงโค้ด TypeScript ทั้งหมดให้ออกมาเป็นไฟล์เว็บบริสุทธิ์ธรรมดา ได้แก่ **HTML (index.html), CSS และ JavaScript แท้** เก็บอยู่ในโฟลเดอร์ชื่อ **`dist`** ซึ่งเป็นไฟล์ที่เว็บเซิร์ฟเวอร์เสถียรทั่วไปเข้าใจและรันได้ทันทีโดยไม่ต้องติดตั้ง Node.js บน Server จริงก็ได้ครับ
   
2. **ฝั่งหลังบ้านและระบบเก็บข้อมูล (Backend & SQL Database):**
   * ระบบใช้สคริปต์ **`obec_pa_connector.php`** รันด้วยภาษา **PHP ตัวจริง 100%** !
   * สคริปต์ตัวนี้ทำหน้าที่เชื่อมต่อกับฐานข้อมูล SQL (`schoolos_Pateacher`) ของคุณครูด้วยระบบความปลอดภัยสูงสุดแบบ **PDO (PHP Data Objects)**
   * มีความพิเศษที่ระบบอัจฉริยะ **Auto-Initialization** เมื่อเรียกไฟล์ PHP นี้ครั้งแรกในบราวน์เซอร์ ระบบจะสร้างตารางประวัติ ตาราง 15 ตัวชี้วัด และตารางไฟล์แนบหลักฐานเข้า phpMyAdmin ให้ครบถ้วนโดยอัตโนมัติ

## 🛠️ วิธีแก้ไขปัญหา Build ล้มเหลวบน Plesk Windows (Access is Denied)

เนื่องจากเซิร์ฟเวอร์ประเภท Windows IIS ร่วมกับ Plesk จะมีข้อจำกัดด้านสิทธิ์การเข้าถึงความปลอดภัย (Permissions) ทำให้เวลาแอปเขียนคำสั่งคอมไพล์ผ่าน Command Line ใน Plesk และย้อนหาพาธชั้นสูงโดนบล็อกสิทธิ์ ทางเลือกระดับมาตรฐานที่ง่าย สะดวก และมั่นคงที่สุด มีขั้นตอนดังนี้ครับ:

### ✅ วิธีที่ 1: Build ในคอมพิวเตอร์ของคุณครูแล้วอัปโหลด (วิธีที่แนะนำและง่ายที่สุด)

วิธีนี้เหมาะสำหรับท่านที่ไม่ต้องการลงหรือแก้สิทธิ์ในเครื่อง Windows Server ส่วนกลาง:

1. **ดาวน์โหลดโปรเจกต์มาทำการในเครื่องคอมพิวเตอร์ส่วนตัวของคุณครู** (โดยดึงเนื้อหาจาก GitHub หรือดาวน์โหลด ZIP จากหน้านี้)
2. เปิดตัวโปรแกรม Terminal/Command Prompt ในเครื่องคอมฯ ของท่าน แล้วพิมพ์คำสั่งเพื่อติดตั้งแพเกจเสริม:
   ```bash
   npm install
   ```
3. สั่งคอมไพล์โค้ดของคุณครูด้วยคำสั่งดังนี้:
   ```bash
   npm run build
   ```
4. ในคอมพิวเตอร์ของคุณครูจะเกิดโฟลเดอร์ใหม่ชื่อ **`dist`** (ซึ่งประกอบไปด้วยไฟล์ `index.html` และโฟลเดอร์ `assets` ที่ผ่านการบีบอัดเรียบร้อยแล้ว)
5. **ทำการอัปโหลดเฉพาะโฟลเดอร์ `dist` และไฟล์เชื่อมต่อ `obec_pa_connector.php`** ขึ้นสู่ Server จริงของคุณครูผ่านหน้าจัดการไฟล์ใน Plesk หรือ FTP ได้เลยครับ โดยวางไว้ในโฟลเดอร์หลักของห้องเว็บ (เช่น `httpdocs` หรือ `PA.schoolos-app.com`)
